# Code of Conduct

This project is a public technical handbook. The expected standard is respectful collaboration, technical honesty, and evidence-backed discussion.

## Scope

This Code of Conduct applies to:

- the GitHub repository
- issues and pull requests
- discussions about repository content or releases
- public project communication spaces controlled by maintainers

## Expected Behavior

Contributors and maintainers are expected to:

- be respectful, direct, and specific
- correct errors without personal attacks
- cite primary sources when making factual claims
- distinguish tested guidance from unverified guidance
- accept review feedback in good faith
- keep security concerns high-signal and private where appropriate

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment, threats, or discriminatory language
- personal attacks, dogpiling, or intimidation
- knowingly adding hallucinated commands or unsupported claims
- misrepresenting vendor capabilities, pricing, or security posture
- posting real secrets, private code, customer data, or exploit details in public threads
- spamming, trolling, or derailing technical review

## Maintainer Responsibilities

Maintainers are responsible for:

- setting and enforcing this Code of Conduct
- prioritizing contributor safety and repository integrity
- removing unsafe or misleading technical guidance
- keeping moderation actions proportionate to the behavior

## Reporting

For routine community issues, open a GitHub issue or discussion if one is available.

For security-sensitive conduct or content concerns, follow [SECURITY.md](./SECURITY.md) and avoid public disclosure of exploit details, secrets, or private data.

If GitHub private reporting is not available, open a minimal public issue requesting a private contact path without posting sensitive content.

## Enforcement

Maintainers may:

- edit or remove unsafe content
- close or lock threads
- reject or revert contributions
- temporarily or permanently restrict participation

## Enforcement Guidelines

Examples of proportional responses:

- correction request for factual inaccuracies
- warning for repeated low-signal or disrespectful behavior
- contribution rejection for unsafe or misleading technical content
- temporary restriction for harassment or repeated policy violations
- permanent ban for severe abuse, malicious behavior, or deliberate credential exposure

## Confidentiality

Maintainers will handle reports as discreetly as practical, but absolute confidentiality cannot be promised on a public GitHub project. Sensitive material should not be posted publicly.

## Attribution

This policy is informed by the Contributor Covenant approach, but adapted for a technical documentation project that emphasizes verification, safety, and accuracy.
