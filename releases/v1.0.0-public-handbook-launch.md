# v1.0.0 - Public Handbook Launch

## Highlights

- launch-ready handbook structure for modern AI coding tools, prompts, templates, MCP workflows, and team governance
- GitHub Pages documentation site built with HonKit
- stronger validation pipeline for links, navigation, metadata, Markdown, build output, and forbidden local paths
- improved contribution, security, and review standards for public collaboration

## Included in This Release

- tool guides for terminal, IDE, and agent-first coding tools
- config-file guidance for `AGENTS.md`, `CLAUDE.md`, Cursor rules, Copilot instructions, and MCP configs
- workflow starters for review, debugging, upgrades, testing, release work, and documentation
- MCP evaluation guidance with least-privilege framing
- prompt starters and master prompts for recurring engineering tasks
- templates for team adoption, evaluation, and review
- governance starters for rollout, approvals, prompt review, and security controls

## Validation System

- pinned HonKit and Node/npm versions for reproducible builds
- GitHub Actions workflows for docs validation, Markdown linting, link checks, and Pages deploy
- internal-link, anchor, and generated-route validation
- local filesystem path detection for public repository files

## Known Limitations

- some tools and MCP implementations remain `Documentation verified`, `Requires account`, or `Needs verification`
- enterprise-plan, paid-plan, and host-specific behavior is not always locally testable
- governance pages are policy starters and must be customized by each engineering organization

## Recommended Next Steps

- keep tool install and compatibility notes current against official docs
- continue converting thin workflow and MCP pages into deeper verified guides
- add company-fork examples for Java, Angular, and full-stack teams
