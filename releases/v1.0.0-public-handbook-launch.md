# v1.0.0 - Public Handbook Launch

This is the first public release of **AI Tools Cheatsheets**, a vendor-neutral handbook for modern AI coding tools, coding agents, MCP workflows, prompts, templates, and team governance.

## Highlights

- tool guides for Claude Code, OpenAI Codex, Cursor, GitHub Copilot, Gemini CLI, Grok/xAI, and open-source coding tools
- a tool-selection matrix and getting-started guidance for individual developers and teams
- repository instruction-file guidance for `AGENTS.md`, `CLAUDE.md`, Cursor rules, Copilot instructions, and MCP configs
- engineering workflow playbooks for code review, bug fixing, CI debugging, testing, migrations, and release work
- MCP evaluation guidance with least-privilege framing and approval-focused checklists
- prompt starters and reusable master prompts for common engineering tasks
- templates and governance starters for internal team playbooks
- GitHub Pages documentation site built with HonKit
- automated documentation validation for paths, navigation, metadata, Markdown, build output, and links
- stronger public contribution infrastructure with issue forms, PR template, Code of Conduct, and Security Policy

## Verification Philosophy

- official sources are preferred wherever possible
- verification dates and status labels are explicit
- documentation-only verification is labeled as such
- untested or account-gated behavior is not presented as locally tested
- MCP integrations are treated as privileged systems
- human review remains mandatory for merge, deploy, destructive shell actions, and privileged access changes

## Known Limitations

- some account-gated or paid-plan features remain `Requires account`, `Requires paid plan`, or `Not locally tested`
- some MCP pages are evaluation checklists rather than verified installation guides
- governance templates are policy starters and require company-specific review
- fast-moving tools and vendor surfaces will require periodic re-verification
- some external link checks may warn on network timeout or rate-limit behavior even when internal validation passes
