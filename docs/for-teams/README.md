# For Teams

Use this page if your team wants to standardize AI coding tools, prompts, repository instructions, MCP usage, and contribution practices without losing security or developer autonomy.

## Who This Is For

- engineering teams adopting coding agents together
- developer-productivity and DX teams
- maintainers building internal AI engineering handbooks

## Best First Steps

1. Pick a default evaluation flow in [Choosing the Right Tool](../getting-started/choosing-the-right-tool.md).
2. Standardize repo instructions with [AGENTS.md](../configs/agents-md.md).
3. Review [Security and Permissions](../governance/security-and-permissions.md).
4. Use [Team Rollout Guide](../governance/team-rollout-guide.md) to phase adoption.
