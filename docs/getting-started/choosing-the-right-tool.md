# Choosing the Right Tool

## Use Claude Code

For deep multi-file coding tasks, refactoring, architecture work, and terminal-first repo analysis.

## Use OpenAI Codex

For terminal-first OpenAI coding workflows, `AGENTS.md`-based repo guidance, code review, and agentic development across CLI and cloud surfaces.

## Use Cursor

For developers who want AI deeply integrated into their IDE.

## Use GitHub Copilot

When the team already works inside GitHub and VS Code or JetBrains.

## Use Grok / xAI

As a model or integration option unless official CLI support is confirmed.

## Use Cline, Roo Code, Continue, Aider, Or OpenCode

When the team wants open-source or editor-based alternatives with more provider flexibility.

> Verification note: Treat xAI as a model/integration path unless official first-party CLI guidance is confirmed in xAI docs.

> Team tip: Most teams should standardize one primary terminal tool and one primary IDE tool rather than approving every option.
