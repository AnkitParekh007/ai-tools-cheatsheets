# Why This Repo Exists

AI coding docs are scattered across vendor sites, release notes, forum posts, prompt snippets, and screenshots. That makes it hard for developers and teams to answer basic questions consistently.

## Why this repo exists

- to keep AI coding tool guidance in one place
- to make verified commands easy to copy
- to help teams compare tools before standardizing
- to make governance, prompts, and templates reusable

## Who should use it

- developers evaluating tools
- engineering leads choosing defaults
- platform and developer productivity teams
- contributors who want to improve shared tooling docs

## How developers benefit

- faster onboarding
- less guesswork on commands and setup
- reusable prompts and workflows

## How teams benefit

- forkable internal handbook
- reusable governance pages
- templates for repo instructions and approvals

## How contributors benefit

- clear contribution surfaces
- verified-docs impact
- public open-source documentation work

## Why verification matters

> Verification note: AI coding tools change quickly. This repo prefers official docs, clear status markers, and dated verification over guesswork.
