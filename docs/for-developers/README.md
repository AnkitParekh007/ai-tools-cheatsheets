# For Developers

Use this page if you want to choose an AI coding tool, configure a repository, adopt safer daily workflows, and find the fastest path to practical examples.

## Who This Is For

- individual developers comparing coding agents
- frontend, backend, QA, and DevOps engineers
- contributors trying a new tool in an existing repository

## Best First Steps

1. Read [Choosing the Right Tool](../getting-started/choosing-the-right-tool.md).
2. Compare options in the [Comparison Matrix](../getting-started/comparison-matrix.md).
3. Set shared repo instructions with [AGENTS.md](../configs/agents-md.md) or [CLAUDE.md](../configs/claude-md.md).
4. Adopt one safe workflow such as [Code Review](../workflows/code-review.md) or [Bug Fixing](../workflows/bug-fixing.md).

## Quick Checklist

- choose one primary tool before adding more
- define repo instructions before running broad edits
- prefer read-only or low-risk workflows first
- keep human review for merge, deploy, and permission changes
