# For Engineering Leaders

Use this page if you need a practical starting point for evaluating AI coding tools as an engineering capability, not just as isolated individual experiments.

## Who This Is For

- engineering managers
- heads of engineering
- CTOs
- developer-productivity leaders

## What You Can Use This Handbook For

- compare tool categories without vendor lock-in
- define secure rollout phases
- create internal evaluation and approval criteria
- decide where prompts, MCP, and repo instructions fit into engineering standards

## Best First Steps

1. Review the [Comparison Matrix](../getting-started/comparison-matrix.md).
2. Read [Team Rollout Guide](../governance/team-rollout-guide.md).
3. Use [Approved Tools Policy](../governance/approved-tools-policy.md) and [Tool Evaluation Template](../templates/tool-evaluation-template.md).
4. Review [MCP Approval Policy](../governance/mcp-approval-policy.md) before allowing privileged integrations.
