# Audience and Search Intent

Audit date: `2026-07-15`

## Developer Segments

### AI-Curious Developer

- Problems: choosing a tool, setting it up safely, finding good first workflows
- Search queries: `best AI coding tool`, `Claude Code vs Codex`, `AI code review prompt`
- Preferred formats: comparisons, quick starts, prompts, templates
- Best current pages: `Comparison Matrix`, `Choosing the Right Tool`, `For Developers`
- Missing pages: head-to-head comparisons and stack-specific tool pages
- Desired next action: open the docs site, pick one tool, then follow a workflow page

### Frontend or Angular Engineer

- Problems: upgrades, UI workflows, IDE vs terminal fit
- Search queries: `best AI coding tools for Angular`, `AI Angular upgrade workflow`
- Preferred formats: workflow guides, prompts, comparison summaries
- Best current pages: `Angular Workflows`, `Cursor`, `Claude Code`, `For Developers`
- Missing pages: Angular-specific `AGENTS.md` examples and tool comparisons
- Desired next action: adopt one workflow and one repo-instruction file

### Backend or Java/Spring Engineer

- Problems: debugging layered services, planning safe fixes, repo-wide reasoning
- Search queries: `AI Spring debugging`, `AI Java code review`, `best AI coding tool for backend`
- Preferred formats: workflow guides, prompt starters, policy-aware setup docs
- Best current pages: `Java Spring Workflows`, `Code Review`, `AGENTS.md`
- Missing pages: Java/Spring repository examples
- Desired next action: use a stack-specific workflow and prompt starter

### QA Engineer

- Problems: generating tests, reviewing behavior changes, validating prompts
- Search queries: `AI test generation workflow`, `AI regression test prompt`

### DevOps or Platform Engineer

- Problems: CI debugging, permissions, repo-wide automation, safe rollout
- Search queries: `AI CI/CD debugging`, `secure coding agents`, `MCP approval policy`

### Security Engineer

### Engineering Lead

## Organization Segments

### Engineering Manager

- Business concern: tool sprawl and inconsistent standards
- Search intent: `AI coding rollout plan`, `AI coding tools for teams`
- Required evidence: rollout steps, approval policies, templates
- Decision criteria: adoption effort, validation process, review quality
- Relevant templates: `AI Task Brief Template`, `Tool Evaluation Template`
- Desired next action: review `For Teams`

### Head of Engineering

- Business concern: adoption risk vs productivity upside
- Search intent: `enterprise AI coding adoption`, `AI coding governance`
- Required evidence: evaluation criteria, metrics, security controls
- Decision criteria: governance readiness, team fit, reversibility
- Relevant templates: `Approved Tools Policy`, `Team Rollout Guide`
- Desired next action: review `For Engineering Leaders`

### CTO

- Business concern: long-term architecture, platform risk, internal leverage
- Search intent: `internal AI engineering handbook`, `AI developer productivity strategy`
- Required evidence: neutral comparison, governance, reusable operating model
- Decision criteria: strategic flexibility, security posture, team standardization
- Relevant templates: governance and rollout pages
- Desired next action: approve an internal fork or pilot

### Developer-Experience Leader

- Business concern: shared prompts, templates, repo instructions, training
- Search intent: `AI coding standards`, `AGENTS.md template`, `shared AI prompts`
- Required evidence: copy-ready files, workflow docs, contribution model
- Decision criteria: portability, clarity, adoption friction
- Relevant templates: `AGENTS.md`, `CLAUDE.md`, prompt library
- Desired next action: roll out repo instructions and prompt standards

### Platform Engineering Team

- Business concern: integrations, permissions, deployment risk, supportability
- Search intent: `secure coding agents`, `MCP approval`, `AI tool permissions`
- Required evidence: least privilege, revocation, pilot path
- Decision criteria: read/write scope, operational support, auditability
- Relevant templates: `Security Checklist Template`, `MCP Approval Policy`
- Desired next action: run a read-only pilot

### Security and Compliance Team

- Business concern: credentials, private code, unsafe automation, external access
- Search intent: `AI coding security policy`, `MCP security checklist`
- Required evidence: explicit controls, incident response, owner assignment
- Decision criteria: risk tiers, revocation, human approval gates

### Procurement or Tool-Evaluation Team
